# 前端 Web 模块规则

## 1. 模块职责

| 项 | 内容 |
|----|------|
| 模块范围 | `cortex-frontend/src` |
| 核心职责 | 页面路由、任务/项目可视化、交互与请求编排 |
| 技术栈 | React + React Query + Ant Design |

## 2. 分层边界

| 层级 | 目录 | 职责 |
|------|------|------|
| 入口层 | `main.tsx`、`App.tsx` | 应用装配、路由守卫、页面路由 |
| 页面功能层 | `features/*` | 按业务域组织页面和服务 |
| 组件层 | `components/*` | 跨页面可复用 UI 组件 |
| 基础设施层 | `lib/*` | HTTP 封装、QueryClient 配置 |
| 共享层 | `constants`、`types`、`utils` | 常量、类型、格式化与展示工具 |

## 3. 数据来源

| 数据 | 来源 |
|------|------|
| 认证与用户信息 | 后端 API |
| 任务/项目/评论数据 | 后端 API |
| 登录态 Token | `localStorage` |
| 页面缓存 | React Query |

## 4. 关键机制

| 机制 | 说明 |
|------|------|
| 鉴权机制 | 路由守卫 + Axios 请求拦截器附加 Token |
| 请求机制 | 统一由 `lib/http.ts` 发起 |
| 缓存机制 | 通过 Query Key 管理缓存与失效 |
| 看板机制 | 拖拽更新任务状态并回写后端 |

## 5. 依赖关系

| 依赖模块 | 关系 |
|----------|------|
| 后端 API 模块 | 强依赖，所有业务数据来自 API |
| 业务配置规范 | 状态、优先级展示需对齐统一配置 |

## 6. 变更规则

1. 新增页面或功能时，优先落在对应 `features/<domain>`。
2. 通用常量与格式化逻辑必须提取到 `constants`/`utils`。
3. API 请求必须复用统一 HTTP 实例，不可直接散落 `fetch`。

## 7. 严禁事项

- ⚠️ 严禁在多个页面重复定义任务状态/优先级映射。
- ⚠️ 严禁在组件层直接硬编码后端地址。
- ⚠️ 严禁将鉴权 Token 写入日志或 URL 查询参数。
