# CLI 工作流模块规则

## 1. 模块职责

| 项 | 内容 |
|----|------|
| 模块范围 | `cortex-backend/cli` |
| 核心职责 | 任务状态流转、Git 分支/PR 自动化、AI 辅助 |
| 入口命令 | `ctx`（Typer） |

## 2. 命令边界

| 命令组 | 文件 | 职责 |
|--------|------|------|
| `auth` | `commands/auth.py` | 登录、登出 |
| `tasks` | `commands/tasks.py` | 任务创建、开始、删除、提交流程、完成 |
| `config` | `commands/config.py` | CLI 配置查询与设置 |
| `review` | `commands/review.py` | AI 审查执行与状态查询 |

## 3. Provider 机制

| 类型 | 目录 | 职责 |
|------|------|------|
| Git Provider | `providers/*.py` | 创建/查询/合并 PR 或 MR |
| PR 评论 Provider | `providers/pr_comment/*.py` | 将审查结果回写评论 |

## 4. 数据来源

| 数据 | 来源 |
|------|------|
| 任务与用户信息 | 后端 API |
| 当前分支状态 | 本地 Git 仓库 |
| 平台 PR/MR 信息 | Git 平台 API（Provider） |
| AI 结果 | `cli/ai` 子模块（配置驱动） |

## 5. 关键机制

| 机制 | 说明 |
|------|------|
| 状态流转机制 | `start/pr/done` 与任务状态绑定 |
| 删除机制 | `delete` 命令通过后端任务 API 执行软删除，删除前必须显式确认 |
| 分支机制 | 按任务生成/识别分支名并驱动 Git 操作 |
| 配置机制 | 本地 CLI 配置控制 provider 与 AI 行为 |
| 审查机制 | 支持生成评审内容并写入平台评论，并将审查摘要同步到任务评论 |
| 代理兼容机制 | 当 AI 请求走 SOCKS 代理时，CLI 运行环境必须包含 `httpx[socks]`；依赖缺失时 CLI 只输出明确失败提示，不应继续回写误导性审查评论 |

## 6. 依赖关系

| 依赖模块 | 关系 |
|----------|------|
| 后端 API 模块 | 强依赖，读取和更新任务状态 |
| 运行与部署模块 | 命令执行依赖本地 Git 与环境配置 |
| Git 平台 | 外部依赖，负责 PR/MR 生命周期 |

## 7. 变更规则

1. 修改任务状态流转时，必须与后端状态定义对齐。
2. 修改分支命名规则时，必须同步校验 `start/pr/done/review` 全流程。
3. 新增高风险任务命令时，必须补齐确认步骤与失败提示。
4. 引入新 provider 时，必须补齐工厂注册与配置项说明。

## 8. 严禁事项

- ⚠️ 严禁在状态未同步成功时继续执行后续自动化步骤且不报错。
- ⚠️ 严禁在命令输出中打印密钥、Token。
- ⚠️ 严禁新增 provider 但不更新配置键与文档。
