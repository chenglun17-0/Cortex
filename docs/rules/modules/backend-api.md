# 后端 API 模块规则

## 1. 模块职责

| 项 | 内容 |
|----|------|
| 模块范围 | `cortex-backend/app` |
| 核心职责 | 提供认证、组织、项目、任务、相似检索 API |
| 对外接口 | `/api/v1/*` |

## 2. 子模块边界

| 子模块 | 目录 | 职责 |
|--------|------|------|
| 应用装配 | `app/main.py` | 注册路由与数据库连接 |
| API 接入层 | `app/api/v1/endpoints` | HTTP 入参、鉴权依赖、响应返回 |
| 数据模型层 | `app/models` | Tortoise ORM 模型定义 |
| 契约层 | `app/schemas` | Pydantic 请求/响应结构 |
| 服务层 | `app/services` | 向量检索等可复用业务服务 |
| 核心能力 | `app/core` | 配置、安全与上下文能力 |

## 3. 数据来源

| 数据 | 来源 |
|------|------|
| 用户/组织/项目/任务主数据 | PostgreSQL（Tortoise ORM） |
| 认证信息 | JWT + 用户表 |
| 相似检索数据 | pgvector 向量表 |

## 4. 关键机制

| 机制 | 说明 |
|------|------|
| 鉴权机制 | `Authorization: Bearer` + `get_current_user` |
| 路由机制 | 统一通过 `app/api/v1/api.py` 聚合注册 |
| 软删除机制 | 任务/项目查询需过滤 `deleted_at` |
| 向量检索机制 | 任务变更时同步写入/删除 embedding |

## 5. 依赖关系

| 依赖模块 | 关系 |
|----------|------|
| 数据库与向量模块 | 强依赖，负责存储和检索 |
| CLI 模块 | CLI 调用 API 同步任务状态 |
| 前端模块 | 前端页面直接消费 API |

## 6. 变更规则

1. 新增/修改 API 时，必须同步更新对应 `schemas`。
2. 涉及数据库结构变更时，必须同步维护 migration。
3. 涉及任务状态变更时，必须检查前端与 CLI 联动逻辑。
4. 生产环境必须关闭自动建表；当前代码若启用自动建表，只允许本地开发环境使用。

## 7. 执行检查

```bash
# 检查是否开启自动建表（必须结合环境判断）
rg -n "generate_schemas\\s*=\\s*True|generate_schemas\\s*=\\s*False" cortex-backend/app/main.py
```

## 8. 严禁事项

- ⚠️ 严禁绕过鉴权依赖直接暴露用户敏感数据。
- ⚠️ 严禁在路由层堆积不可复用的复杂逻辑，公共逻辑应下沉服务层。
- ⚠️ 严禁在生产环境依赖自动建表行为。
